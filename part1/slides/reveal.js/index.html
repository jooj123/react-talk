What is React?

  • A Javascript Library for creating user interfaces
  • Only a "View" Library - makes no assumptions about the rest of your tech stack. Makes it easier to integrate with existing projects.
  • Used by Facebook, Airbnb, Apple, Uber and many more

Advantages

  • Virtual DOM: abstracts Document Object Model (DOM) for better performance
  • Component driven development (easier and more maintainable development)
  • Server side rendering via Node.js (performance and SEO benefits)

Components

React is all about building reusable components

  • Easier to test
  • Composability
  • Take input data (props)
  • "render" method to define what to display in a XML / HTML like syntax called JSX
  • Can have changing state

Show simple component demo

Component Lifecycle

  • componentWillMount() - before initial render
  • render() - examine this.props and this.state and return a virtual representation of a native DOM component (eg. div) or another composite component that you've defined yourself (eg. HelloWorld)
  • componentDidMount() - when initial rendering occurs, can access DOM representation if needed (eg. 3rd party libs)

Component Lifecycle

  • componentWillReceiveProps(object nextProps) - when a component is receiving new props
  • shouldComponentUpdate(object nextProps, object nextState) - not called for initial render, used for optimisations
  • componentWillUpdate() - opportunity to perform preparation before an update occurs
  • componentDidUpdate(object prevProps, object prevState) - component's updates are flushed to the DOM
  • componentWillUnmount() - used for cleanup
